Construyendo Clases Reutilizables en Python

Una guía práctica para Ciencia de Datos basada en la Programación Orientada a Objetos.

Crear código que "funcione" es solo el primer paso. Para ser eficientes y colaborativos en ciencia de datos, necesitamos construir herramientas que sean legibles, mantenibles y reutilizables. La Programación Orientada a Objetos (POO) nos ofrece el paradigma perfecto para lograrlo.

Esta guía, inspirada en los conceptos de tu documento, te mostrará cómo modelar problemas a través de clases y objetos para crear una potente herramienta de Análisis Exploratorio de Datos (EDA).

1. Los Pilares de la Programación Orientada a Objetos (POO)

1.1. ¿Qué es una Clase y qué es un Objeto?

La relación entre una clase y un objeto es la misma que existe entre un plano arquitectónico y la casa construida a partir de él.

  • Clase: Es el plano, el molde o la plantilla. Define la estructura (atributos) y los comportamientos (métodos) que tendrán los objetos de un tipo particular.
  • Objeto (o Instancia): Es la casa. Una manifestación concreta y única creada a partir de una clase. Cada objeto tiene su propio estado (sus atributos tienen valores específicos), pero comparte los mismos comportamientos definidos en la clase.

1.2. Anatomía de una Clase: `__init__`, Atributos, Métodos y `self`

El Método `__init__`: El Inicializador del Objeto

El método __init__ es, quizás, el más importante de una clase. Es un "método mágico" (reconocible por los dobles guiones bajos) que Python ejecuta automáticamente cada vez que se crea una nueva instancia (un objeto) de la clase. Su principal y única responsabilidad es inicializar el estado del objeto, es decir, asignar los valores iniciales a sus atributos (self.atributo = valor).

Aunque a menudo se le llama "constructor" (por su similitud con otros lenguajes como Java o C++), técnicamente en Python su rol no es crear el objeto, sino configurarlo una vez que ya ha sido creado en memoria. Por esta razón, __init__ nunca debe retornar un valor.

Con el rol de `__init__` claro, desglosemos los demás componentes principales:

  • Atributos: Son las variables que pertenecen a un objeto. Almacenan el estado del objeto. En nuestro ejemplo, `self.datos` es un atributo que guarda el DataFrame que queremos analizar. Se definen típicamente en el método `__init__`.
  • Métodos: Son las funciones que pertenecen a una clase. Definen el comportamiento que los objetos pueden realizar. `resumen_descriptivo()` es un método que sabe cómo imprimir un resumen de los datos almacenados en el atributo `self.datos`.
  • El parámetro `self`: Es la pieza clave que conecta los métodos con los atributos de una instancia específica. `self` es una referencia al objeto mismo. Cuando llamas a `explorador_titanic.resumen_descriptivo()`, Python automáticamente pasa el objeto `explorador_titanic` como el primer argumento, `self`. Esto permite que el método acceda a `self.datos` (el DataFrame del Titanic) y no al de otro objeto.
  • Docstrings (Cadenas de Documentación): Son cadenas de texto literales que aparecen justo después de la definición de un módulo, función, clase o método. Se encierran entre triples comillas `""" """` y son la forma estándar de documentar el código en Python. Explican qué hace el código, qué parámetros recibe y qué devuelve. Son fundamentales para la legibilidad y el mantenimiento. 
class ExploradorDatos:
    """
    Este es el docstring de la CLASE.
    Explica el propósito general de ExploradorDatos.
    """
    def __init__(self, dataframe: pd.DataFrame):
        """
        Este es el docstring del MÉTODO __init__.
        Explica que inicializa el objeto y qué parámetro espera.
        """
        # 'self.datos' es un ATRIBUTO. 'self' lo vincula a la instancia.
        self.datos = dataframe

    def resumen_descriptivo(self):
        """Este es el docstring de otro MÉTODO."""
        # El método accede al ATRIBUTO a través de 'self'.
        print(self.datos.describe())

2. Caso Práctico: Clase `ExploradorDatos` Completa

Ahora, con los conceptos claros, presentamos el código completo y documentado de nuestra clase. Este código debería guardarse en un archivo llamado `analisis_eda.py` para poder importarlo en nuestros notebooks. 

# analisis_eda.py
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

class ExploradorDatos:
    """
    Una clase para realizar un Análisis Exploratorio de Datos (EDA) básico.
    
    Permite obtener resúmenes, y visualizar análisis univariados y bivariados
    de un conjunto de datos proporcionado como un DataFrame de pandas.

    Atributos
    ----------
    datos : pd.DataFrame
        El DataFrame que se va a analizar.
    """

    def __init__(self, dataframe: pd.DataFrame):
        """
        Inicializa el objeto ExploradorDatos.

        Parámetros
        ----------
        dataframe : pd.DataFrame
            El conjunto de datos para el análisis.
        """
        if not isinstance(dataframe, pd.DataFrame):
            raise TypeError("El input debe ser un DataFrame de pandas.")
        self.datos = dataframe
        print("Objeto ExploradorDatos creado con éxito.")

    def resumen_descriptivo(self):
        """Imprime un resumen completo del DataFrame."""
        print("--- Información General ---")
        self.datos.info()
        print("\n", "--- Estadísticas Descriptivas (Numéricas) ---")
        print(self.datos.describe())
        print("\n", "--- Conteo de Valores Nulos ---")
        print(self.datos.isnull().sum())

    def analisis_univariado(self, columna: str):
        """Genera un gráfico univariado para una columna específica."""
        if columna not in self.datos.columns:
            print(f"Error: La columna '{columna}' no existe.")
            return
        
        plt.style.use('seaborn-v0_8-whitegrid')
        plt.figure(figsize=(10, 6))
        
        if pd.api.types.is_numeric_dtype(self.datos[columna]):
            sns.histplot(self.datos[columna], kde=True)
            plt.title(f'Distribución de {columna}', fontsize=16)
        else:
            sns.countplot(x=columna, data=self.datos, order=self.datos[columna].value_counts().index)
            plt.title(f'Conteo de categorías en {columna}', fontsize=16)
            plt.xticks(rotation=45)
        
        plt.ylabel('Frecuencia')
        plt.xlabel(columna)
        plt.tight_layout()
        plt.show()

    def analisis_bivariado(self, var_explicativa: str, var_respuesta: str):
        """
        Genera un gráfico bivariado entre dos variables.
        - Categórica vs Numérica: Boxplot
        - Categórica vs Categórica: Gráfico de barras agrupado
        """
        for col in [var_explicativa, var_respuesta]:
            if col not in self.datos.columns:
                print(f"Error: La columna '{col}' no existe.")
                return
        
        plt.style.use('seaborn-v0_8-whitegrid')
        plt.figure(figsize=(12, 8))
        
        if not pd.api.types.is_numeric_dtype(self.datos[var_explicativa]) and pd.api.types.is_numeric_dtype(self.datos[var_respuesta]):
            sns.boxplot(x=var_explicativa, y=var_respuesta, data=self.datos)
            plt.title(f'{var_respuesta} por {var_explicativa}', fontsize=16)
            plt.xticks(rotation=45)
        elif not pd.api.types.is_numeric_dtype(self.datos[var_explicativa]) and not pd.api.types.is_numeric_dtype(self.datos[var_respuesta]):
            sns.countplot(x=var_explicativa, hue=var_respuesta, data=self.datos)
            plt.title(f'Distribución de {var_explicativa} por {var_respuesta}', fontsize=16)
            plt.xticks(rotation=45)
        else:
            print("Combinación de tipos no soportada. Prueba Categórica vs Numérica o Categórica vs Categórica.")
            return
            
        plt.tight_layout()
        plt.show()

3. ¿Cómo Usar Nuestra Clase en un Notebook?

Una vez guardado el archivo `analisis_eda.py`, podemos importarlo y usarlo en cualquier notebook (como Jupyter o Google Colab) que esté en el mismo directorio.

Vamos a usar el famoso dataset del Titanic como ejemplo:

# 1. Importar librerías y nuestra nueva clase
from analisis_eda import ExploradorDatos
import seaborn as sns

# 2. Cargar un conjunto de datos
df_titanic = sns.load_dataset('titanic')

# 3. Crear un objeto de nuestra clase
explorador_titanic = ExploradorDatos(df_titanic)

# 4. Usar los métodos del objeto
explorador_titanic.resumen_descriptivo()
explorador_titanic.analisis_univariado('age')
explorador_titanic.analisis_bivariado('class', 'survived')

4. Buenas Prácticas y Consideraciones

Nombres Descriptivos

Nombres como `ExploradorDatos` o `analisis_univariado` comunican claramente su propósito y hacen el código más legible.

Principio DRY (No te Repitas)

Al encapsular la lógica, evitamos duplicar código. Si queremos cambiar los gráficos, solo editamos el método correspondiente una vez.

Documentación (Docstrings)

Los docstrings son esenciales para que otros (y tú en el futuro) puedan entender y usar tu clase sin tener que descifrar el código.

Cuidado con Datos Grandes

Nuestra clase carga todo el DataFrame en memoria. Para datasets muy grandes (gigabytes), se necesitarían estrategias más avanzadas como el procesamiento en lotes (batch processing).
