# Seminario de Programación: Backend para Ciencia de Datos

## ¿Por qué este curso? La "Última Milla" de la Analítica

Bienvenido al curso de Seminario de Programación de la Maestría en Inteligencia de Negocios.

En el ecosistema actual de los datos, existe un problema común conocido como el "**Notebook Deadlock**". Un Científico de Datos crea un modelo predictivo brillante en su computador, logra un accuracy del 98% y genera visualizaciones impactantes en un Jupyter Notebook. Sin embargo, cuando la gerencia pregunta: "*¿Cómo integramos esto en nuestra app de ventas en tiempo real?*" o "*¿Cómo hacemos que este modelo corra automáticamente cada noche sin fallar?*", la respuesta suele ser silencio.

El valor de negocio real no reside en el modelo en sí, sino en su capacidad de ser **consumido, escalado y mantenido**.

Este curso no se trata de aprender a programar desde cero (eso ya lo sabes). Se trata de **profesionalizar tu código**. Aprenderás a pensar como un Arquitecto de Software para construir las "tuberías" (APIs) que llevan la inteligencia de tus modelos al mundo real.

## Lo que vas a aprender

El objetivo es transformar tu perfil profesional mediante cuatro pilares fundamentales:

1. **Entornos Robustos**: Dejarás atrás el caos de `pip install` y conflictos de versiones. Aprenderás a gestionar dependencias de forma determinista con **Poetry**.

2. **Calidad de Software**: Escribirás código defensivo, modular y testeable (**Unit Testing & OOP Avanzado**), garantizando que tus aplicaciones no se rompan en producción.

3. **APIs Modernas**: Dominarás **FastAPI** (el estándar actual de la industria) para crear servicios web rápidos, documentados y seguros.

4. **Despliegue (Deployment)**: Entenderás cómo "empaquetar" tu solución en contenedores Docker, listos para vivir en la nube.

## Estructura del Contenido

El curso está dividido en 4 módulos intensivos distribuidos en 12 sesiones:

🏗️ **Módulo 1: Ingeniería de Software para Datos**

+ Gestión moderna de entornos (Poetry vs Conda).
+ Programación Orientada a Objetos aplicada a Datos.
+ Clean Code y Test Driven Development (TDD).

🌐 **Módulo 2: Fundamentos de Conectividad**

+ Arquitectura Cliente-Servidor.
+ Protocolo HTTP y RESTful APIs.
+ Consumo programático de datos externos.

🚀 **Módulo 3: Construcción con FastAPI**

+ Creación de APIs de alto rendimiento.
+ Validación estricta de datos con Pydantic.
+ Inyección de dependencias y lógica de negocio.

☁️ **Módulo 4: Integración y Producción**

+ Persistencia de datos con SQLModel (ORM).
+ Autenticación y Seguridad (JWT / OAuth2).
+ Dockerización y estrategias de despliegue en la nube.

## Bibliografía y Referencias

Este curso se fundamenta en literatura técnica de estándares industriales y académicos.

📚 **Lecturas Principales (O'Reilly & CRC Press)**

+ **Percival, H. & Gregory, B. (2020)**. *Architecture Patterns with Python: Enabling Test-Driven Development, Domain-Driven Design, and Event-Driven Microservices*. **O'Reilly Media**.

*Por qué leerlo*: Es la biblia moderna para estructurar aplicaciones Python que sean mantenibles y desacopladas.

+ **Ramalho, L. (2022)**. *Fluent Python: Clear, Concise, and Effective Programming (2nd Edition)*. **O'Reilly Media**.

*Por qué leerlo*: Para entender profundamente el modelo de datos de Python y usar el lenguaje a su máximo potencial.

+ **Voron, F. (2023)**. *Building Data Science Applications with FastAPI*. **Packt Publishing**.

*Por qué leerlo*: Guía específica para conectar modelos de ML con servicios web.

+ **Cook, J. (2017)**. *Docker for Data Science: Building Scalable and Extensible Data Infrastructure*. **CRC Press**.

*Por qué leerlo*: Enfocado en cómo los contenedores resuelven el problema de "en mi máquina funciona".
